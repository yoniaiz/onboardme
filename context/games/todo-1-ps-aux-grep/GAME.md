## Description

This game teaches players to identify services and components from clues about their dependencies and relationships. Players learn to:

- Recognize service patterns
- Understand dependency relationships
- Map services to their purposes
- Connect services to their entry points

## Learning Outcomes

- Understanding project language and framework
- Familiarity with directory structure
- Knowledge of what services exist
- Understanding how to run the project

## Game Flow

Players are given clues about dependencies and must identify which service or component matches those clues. This builds understanding of how services relate to each other and what each service's role is in the system.
